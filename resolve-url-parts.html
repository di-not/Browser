Разбор URL

Определить scheme, host, port, path, query, hash

После того как браузер решил, что ввод похож на URL, он разбирает строку на части: scheme, host, port, path, query, hash. Эти части нужны для дальнейшей навигации: определения origin, проверки безопасности, DNS, выбора соединения и формирования HTTP Request.

Что происходит на этом этапе

  • Браузер получает строку, которую нужно открыть.
  • URL Parser проверяет структуру.
  • Определяется scheme.
  • Определяется host.
  • Определяется port.
  • Определяется path.
  • Определяется query.
  • Определяется hash.
  • На основе этих частей браузер понимает, куда идти дальше.

Разбор на примере

https://example.com:443/products/cars?sort=price#top

Часть URL Значение Зачем нужна
scheme https Определяет протокол и security-контекст.
host example.com Нужен для DNS и определения сервера.
port 443 Указывает порт соединения.
path /products/cars Путь к ресурсу на сервере.
query ?sort=price Параметры запроса.
hash #top Фрагмент внутри страницы, обычно не отправляется на сервер.

Почему это важно

  • host нужен для DNS.
  • scheme влияет на протокол и security.
  • port влияет на соединение.
  • path и query участвуют в HTTP Request.
  • hash обычно не отправляется на сервер.
  • Эти части участвуют в определении origin.

Что часто спрашивают на собеседовании

  • Чем отличается host от origin?
  • Отправляется ли hash на сервер?
  • Что такое default port?
  • Что будет, если scheme не указан?
  • Почему query отправляется на сервер, а hash нет?
