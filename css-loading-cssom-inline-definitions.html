CSS loading и CSSOM

Коротко о блоке

Браузер учитывает стили из нескольких источников: внешней таблицы стилей, элемента <style>, атрибута style="" и базовых стилей браузера. Внешний CSS нужно загрузить отдельным ресурсом; встроенный CSS уже находится в HTML, но его всё равно нужно разобрать. После разбора таблицы стилей представлены в CSSOM, а inline declarations участвуют в каскаде как стили конкретного элемента. 

Есть CSS?
├─ external stylesheet → Загрузка CSS
├─ inline <style>      → CSS parsing
├─ style="..."         → Inline declarations
└─ нет стилей сайта    → Базовые стили браузера

Загрузка CSS
→ CSS parsing
→ CSSOM

CSS loading и CSS parsing подготавливают данные для вычисления стилей. Внешние и встроенные таблицы стилей становятся частью CSSOM; style="" даёт inline declarations; базовые стили браузера применяются даже без CSS, написанного разработчиком.

1. Границы блока

Блок начинается, когда браузер при обработке HTML встречает источник стилей.

Внешняя таблица стилей:

<link rel="stylesheet" href="/styles.css">

Встроенная таблица стилей:

<style>
  h1 {
    color: red;
  }
</style>

Inline style на элементе:

<h1 style="color: red">Title</h1>

Блок заканчивается после подготовки CSS-данных:

CSSOM
+ inline declarations
+ базовые стили браузера

Следующий этап — сопоставить CSS с DOM, применить каскад, получить computed styles и перейти к построению видимого результата. CSSOM не содержит размеры, координаты и пиксели; эти данные появляются позже, на стадиях layout и paint.

2. Внешняя таблица стилей

External stylesheet — внешняя таблица стилей, подключённая к документу через <link rel="stylesheet">. 

<link rel="stylesheet" href="/styles.css">

Элемент <link> связывает документ с внешним ресурсом. При rel="stylesheet" этот ресурс является CSS-таблицей стилей документа.

Путь внешнего CSS:

HTML parser встречает <link rel="stylesheet">
→ браузер берёт URL из href
→ CSS-файл загружается
→ CSS-текст передаётся CSS parser
→ результат участвует в CSSOM

CSS parser — часть браузера, которая разбирает CSS-текст и превращает его во внутренние CSS-структуры.

<link rel="stylesheet"> в <head> по умолчанию блокирует первое корректное отображение страницы: браузеру нужны применимые стили, чтобы построить правильный первый визуальный результат.

При этом HTML parser обычно может продолжать работу: DOM строится дальше, пока CSS загружается. Ограничение касается не самого чтения HTML, а возможности показать страницу без нужных стилей.

Пример с условием применения:

<link rel="stylesheet" href="/main.css">
<link rel="stylesheet" href="/print.css" media="print">

main.css применяется к обычному отображению страницы. print.css предназначен для печати: при просмотре на экране условие media="print" не подходит, поэтому такая таблица стилей не должна задерживать первое отображение страницы на экране.

3. Inline <style>

Inline <style> — встроенная таблица стилей внутри HTML-документа. 

<style>
  body {
    margin: 0;
  }

  .title {
    color: red;
  }
</style>

CSS внутри <style> уже находится в HTML. Для него не нужен отдельный сетевой запрос, но браузер всё равно должен разобрать CSS-текст. 

<style>
→ CSS-текст
→ CSS parsing
→ CSSOM

Такие правила применяются к содержимому документа и участвуют в обычном CSS-процессе: разбираются, попадают в объектное представление и дальше участвуют в каскаде.

4. style=""

style="" — HTML-атрибут, который содержит CSS-декларации прямо на элементе. 

<h1 style="color: red; margin: 0">Title</h1>

Declaration — CSS-декларация, пара property: value, например: 

color: red;

В примере выше у элемента h1 есть две inline declarations:

color: red
margin: 0

Эта ветка не создаёт отдельную таблицу стилей документа. Поэтому style="" не ведёт напрямую в CSSOM как CSSStyleSheet.

Путь inline style:

style=""
→ разбор списка declarations
→ inline declarations элемента
→ участие в cascade

Через HTMLElement.style можно читать и изменять стили, записанные в атрибуте style: 

const title = document.querySelector("h1");

title.style.color = "red";
title.style.margin = "0";

5. CSS parsing

CSS parsing — это переход от потока Unicode-символов к CSS tokens, а затем к CSS objects: stylesheets, rules и declarations. 

stream of Unicode code points
→ CSS tokens
→ CSS objects: stylesheets, rules, declarations

Rule — CSS-правило. В обычном случае оно состоит из selector и блока declarations. 

.card {
  display: grid;
  gap: 16px;
}

Selector — часть правила, которая выбирает элементы документа. В примере это .card.

Declaration — пара property: value. В примере это display: grid и gap: 16px.

Упрощённый результат разбора:

stylesheet
└─ rule
   ├─ selector: .card
   ├─ declaration: display: grid
   └─ declaration: gap: 16px

Для разных источников CSS разбор начинается в разных местах:

<link rel="stylesheet"> → загрузка CSS-файла → parsing stylesheet
<style>                 → parsing stylesheet из HTML
style=""                → parsing declaration list элемента

HTML parsing и CSS parsing — разные процессы:

HTML parsing → DOM
CSS parsing  → CSS rules / declarations / CSSOM

6. CSSOM

CSSOM (CSS Object Model) — объектная модель CSS. Это не “готовые стили элементов” и не визуальное дерево страницы, а представление таблиц стилей и CSS-правил в виде объектов, с которыми может работать браузер и JavaScript API.

Для внешнего stylesheet и для <style> путь выглядит так:

CSS text
→ CSS parsing
→ CSSStyleSheet
→ CSSRuleList
→ CSSRule

CSSStyleSheet — объект, представляющий одну таблицу стилей. Такая таблица может прийти из внешнего файла или из элемента <style>.

После parsing-а браузер может представить эти стили примерно так:

Document
└─ styleSheets
   ├─ CSSStyleSheet from <link rel="stylesheet">
   │  └─ cssRules
   │     ├─ CSSRule
   │     └─ CSSRule
   │
   └─ CSSStyleSheet from <style>
      └─ cssRules
         ├─ CSSStyleRule: h1 { color: red; }
         └─ CSSStyleRule: .card { display: grid; }

На уровне JavaScript список таблиц стилей доступен через:

document.styleSheets

Это не массив DOM-элементов <link> и <style>, а список stylesheet-объектов. Каждый CSSStyleSheet содержит список правил cssRules. 

const sheet = document.styleSheets[0];

sheet.cssRules;

cssRules — список CSS-правил внутри конкретной таблицы стилей.

Обычное CSS-правило:

.card {
  display: grid;
  gap: 16px;
}

Упрощённо в CSSOM:

CSSStyleRule
├─ selectorText: ".card"
└─ style
   ├─ display: "grid"
   └─ gap: "16px"

At-rule:

@media (min-width: 800px) {
  .card {
    display: grid;
  }
}

Упрощённо:

CSSMediaRule
├─ condition: "(min-width: 800px)"
└─ cssRules
   └─ CSSStyleRule: .card { display: grid; }

То есть CSSOM можно представить как структуру stylesheet-ов и правил:

document.styleSheets
→ CSSStyleSheet
→ CSSRuleList
→ CSSRule
→ declarations

Важно: style="" устроен иначе.

<h1 style="color: red; margin: 0">Title</h1>

Атрибут style не создаёт отдельный CSSStyleSheet в document.styleSheets. Он относится к конкретному элементу и доступен через HTMLElement.style. 

const title = document.querySelector("h1");

title.style.color;
title.style.margin;

Поэтому в диаграмме ветка style="" ведёт в inline declarations, а не в CSSOM:

style=""
→ inline declarations конкретного элемента
→ cascade
→ computed styles

CSSOM также не равен DOM.

DOM
→ структура документа:
   html, body, h1, p, button

CSSOM
→ структура CSS:
   stylesheets, rules, selectors, declarations

И CSSOM не равен render tree. В CSSOM ещё нет финального ответа на вопрос, как именно будет выглядеть каждый элемент на экране. 

DOM
+ CSSOM
+ inline declarations
+ базовые стили браузера
→ cascade
→ computed styles
→ render tree

Поэтому CSSOM — это промежуточная модель CSS. Она нужна браузеру, чтобы хранить и обрабатывать таблицы стилей, но финальный внешний вид элемента появляется только после cascade и вычисления computed styles.

7. Базовые стили браузера

Даже если в документе нет внешнего CSS, <style> и style="", браузер применяет базовые стили. 

<h1>Title</h1>
<p>Text</p>
<a href="/about">About</a>

Обычно h1 отображается как заголовок, p как абзац, а a как ссылка. Это результат user-agent styles — стандартных стилей браузера.

В CSS cascade есть несколько источников стилей, или style origins:

user-agent origin
→ базовые стили браузера

author origin
→ стили сайта: <link>, <style>, style=""

user origin
→ стили пользователя, если они есть

Ветка “нет стилей сайта” означает отсутствие author styles. Базовые стили браузера при этом остаются.

8. Cascade и computed styles

После подготовки CSS-данных браузер должен получить итоговые стили для конкретных DOM-элементов. 

DOM
+ CSSOM
+ inline declarations
+ базовые стили браузера
→ cascade
→ computed styles

Cascade — алгоритм выбора итогового значения CSS-свойства, когда к одному элементу подходят несколько деклараций. 

h1 {
  color: black;
}

.title {
  color: red;
}
<h1 class="title">Hello</h1>

К элементу подходят обе декларации. Итоговое значение color выбирается правилами cascade: учитываются origin, importance, cascade layers, specificity, scoping proximity и порядок объявления.

Computed styles — вычисленные значения CSS-свойств после cascade и inheritance. Эти значения используются дальше при построении render tree и последующих стадиях отображения.

9. Связь с DOMContentLoaded

DOMContentLoaded относится к жизненному циклу HTML-документа, а не к отдельной загрузке CSS.

Событие срабатывает после полного parsing HTML и после загрузки/выполнения deferred и module scripts. Оно не ждёт изображения, subframes и async scripts.

Со stylesheets связь косвенная:

defer/module scripts могут ждать stylesheets
→ DOMContentLoaded ждёт defer/module scripts
→ CSS может косвенно задержать DOMContentLoaded

Короткая формула:

DOMContentLoaded ≠ CSS loaded

10. Связь с render

Critical rendering path описывает путь, по которому браузер превращает HTML, CSS и JavaScript в пиксели на экране. Для CSS-блока важна граница: 

CSSOM / inline declarations / базовые стили
→ cascade
→ computed styles
→ render tree

Render tree — дерево видимых узлов с вычисленными стилями. Оно строится позже, на основе DOM и CSS-данных. CSSOM сам по себе не содержит геометрию элементов. 

CSSOM ≠ DOM
CSSOM ≠ render tree
CSSOM ≠ layout
CSSOM ≠ paint

11. Что важно не перепутать

Есть CSS ≠ только внешний CSS-файл

CSS может быть внешним, встроенным в <style> или записанным в style="".

Загрузка CSS относится к external stylesheet

<style> и style="" уже находятся внутри HTML.

style="" ≠ CSSStyleSheet

Inline style относится к конкретному элементу и доступен через HTMLElement.style.

нет стилей сайта ≠ нет стилей вообще

Базовые стили браузера всё равно применяются.

CSSOM ≠ DOM

DOM описывает структуру документа, CSSOM — таблицы стилей и CSS-правила.

CSSOM ≠ render tree

Render tree строится позже на основе DOM и computed styles.

DOMContentLoaded ≠ CSS loaded

CSS может влиять на DOMContentLoaded только косвенно через deferred/module scripts.
