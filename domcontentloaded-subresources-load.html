DOMContentLoaded → subresources → load

Коротко о блоке

Этот блок описывает участок жизненного цикла HTML-документа между событием DOMContentLoaded и событием load. 

DOMContentLoaded
→ Images / subresources загружены?
   ├─ нет → ждать ресурсы, которые задерживают load
   └─ да  → load

DOMContentLoaded означает, что HTML-документ уже разобран и DOM готов для работы. load означает более позднюю точку: страница и зависимые ресурсы, которые участвуют в загрузке страницы, завершили загрузку.

1. Границы блока

Блок начинается событием DOMContentLoaded.

DOMContentLoaded — событие Document. Оно срабатывает после того, как HTML-документ полностью разобран, а deferred scripts и module scripts загружены и выполнены. 

document.addEventListener("DOMContentLoaded", () => {
  console.log("DOM готов");
});

Блок заканчивается событием load.

load для страницы обычно слушают на window. Это событие срабатывает, когда загружена вся страница вместе с зависимыми ресурсами: stylesheets, scripts, iframes, images, кроме ресурсов, которые загружаются лениво. 

window.addEventListener("load", () => {
  console.log("Страница и зависимые ресурсы загружены");
});

2. Что означает DOMContentLoaded

DOMContentLoaded связан с готовностью DOM, а не с полной загрузкой всех ресурсов страницы.

К моменту DOMContentLoaded:

HTML parsing завершён
defer scripts выполнены
module scripts выполнены
DOM уже можно безопасно читать и изменять

Но DOMContentLoaded не ждёт:

images
subframes / iframes
async scripts

Также DOMContentLoaded не является событием завершения загрузки CSS. Но связь со stylesheets возможна косвенно: deferred и module scripts ждут stylesheets, а DOMContentLoaded ставится после выполнения таких scripts.

Поэтому корректная формула:

DOMContentLoaded ≠ все ресурсы загружены
DOMContentLoaded ≠ CSS loaded
DOMContentLoaded ≠ load

3. Что такое subresources

Subresources — ресурсы, которые документ использует дополнительно к основному HTML.

Примеры:

<link rel="stylesheet" href="/styles.css">
<script src="/app.js" async></script>
<img src="/hero.png" alt="">
<iframe src="/widget.html"></iframe>

К подресурсам относятся, например:

stylesheets
scripts
images
iframes
fonts
media resources

В контексте события load важны не “абсолютно все возможные ресурсы”, а ресурсы, которые браузер считает зависимыми ресурсами страницы и которые задерживают завершение загрузки документа. Лениво загружаемые ресурсы, например изображения с loading="lazy", не должны задерживать window.load.

4. Images / subresources загружены?

Узел диаграммы:

Images / subresources загружены?

лучше понимать как проверку:

Завершили ли загрузку ресурсы, которые задерживают load?

Это не означает, что браузер начинает загружать картинки только после DOMContentLoaded. Ресурсы обычно обнаруживаются и загружаются раньше, параллельно с parsing HTML и другими этапами.

Упрощённая временная схема:

HTML parsing идёт
→ браузер обнаруживает images / CSS / scripts / iframes
→ ресурсы начинают загружаться
→ DOMContentLoaded
→ часть ресурсов может ещё загружаться
→ все load-delaying resources завершены
→ load

Изображение может загрузиться до DOMContentLoaded, между DOMContentLoaded и load, или позже, если оно лениво загружается и не участвует в начальной загрузке страницы.

5. Что означает load

load — более позднее событие, чем DOMContentLoaded.

Для страницы window.load означает, что завершилась загрузка страницы и зависимых ресурсов, включая stylesheets, scripts, iframes и images, кроме ресурсов с ленивой загрузкой. 

window.addEventListener("load", () => {
  // можно выполнять код, которому нужны загруженные изображения,
  // iframe или другие зависимые ресурсы
});

load используют, когда код действительно зависит от уже загруженных ресурсов. Например, если нужно работать с изображением после его загрузки.

Но для обычной инициализации интерфейса часто достаточно DOMContentLoaded, потому что ждать все изображения и iframes может быть лишним.

6. readyState: loading, interactive, complete

Состояние документа можно проверить через document.readyState. 

console.log(document.readyState);

Основные значения:

loading
→ HTML ещё загружается и parser работает

interactive
→ HTML разобран, но подресурсы ещё могут загружаться

complete
→ документ и подресурсы загружены, load скоро будет или уже был

Это удобно как ориентир:

loading     → до готовности DOM
interactive → около DOMContentLoaded
complete    → около load

Но readyState не заменяет понимание событий: DOMContentLoaded и load остаются отдельными событиями, которые нужны для разных задач.

7. Что важно не перепутать

DOMContentLoaded ≠ load

DOMContentLoaded наступает раньше. load ждёт больше ресурсов.

DOMContentLoaded ≠ images loaded

Картинки могут ещё загружаться.

DOMContentLoaded ≠ CSS loaded

Событие не ждёт stylesheets напрямую, но CSS может влиять через deferred/module scripts. 

load ≠ все ресурсы, которые когда-либо загрузит страница

Ленивая загрузка, последующие fetch, динамически добавленные ресурсы и действия пользователя могут происходить после load. 

img load ≠ window load

У отдельного <img> может быть своё событие load. window.load относится к загрузке страницы в целом. 

parser complete ≠ DOMContentLoaded

После завершения parsing ещё должны выполниться deferred и module scripts.
